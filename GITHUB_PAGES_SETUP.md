# GitHub Pages Setup Instructions

## Step 1: Enable GitHub Pages ✅

1. Go to: **https://github.com/shweba2/IT-Pulse**
2. Click **Settings** (top menu)
3. Scroll left sidebar → Click **Pages**
4. Under "Build and deployment":
   - **Source**: Select **Deploy from a branch**
   - **Branch**: Select **main**
   - **Folder**: Select **/ (root)**
5. Click **Save**

## Step 2: Wait for Deployment ⏳

GitHub will deploy your site in 1-2 minutes. You'll see a green checkmark next to "Your site is live at..."

## Step 3: Visit Your Site 🎉

Your website will be live at:
**https://shweba2.github.io/IT-Pulse/**

## Step 4: First Automated Update ⏰

The GitHub Actions workflow will automatically run:
- **First run**: Tomorrow at 18:00 UTC (00:30 Yangon time)
- **Frequency**: Every 24 hours
- **Action**: Fetches news from 6 RSS feeds, updates index.html, commits & pushes

## Verify Everything Works

Check that:
- ✅ Site loads at your GitHub Pages URL
- ✅ All cards are uniform (no featured card)
- ✅ Footer shows: "Generated by shweba2 · Public RSS feeds only · No tracking · No cookies"
- ✅ 60 articles display with Yangon timestamps (MMT)

## Files on GitHub

Your repository contains:

```
IT-Pulse/
├── .github/workflows/
│   └── update-news.yml         ← Daily automation (runs at 18:00 UTC)
├── fetch_news.py               ← Python script (fetches RSS feeds)
├── template.html               ← HTML template (uniform cards)
├── requirements.txt            ← Dependencies (requests, feedparser, Jinja2)
├── README.md                   ← User guide
└── index.html                  ← Generated output (updated daily)
```

## How It Works

1. **GitHub Actions runs daily** at 18:00 UTC
2. **Checks out your repo** on a Linux server
3. **Installs Python 3.11** (GitHub's Python, not yours)
4. **Installs dependencies** from requirements.txt
5. **Runs fetch_news.py** which:
   - Fetches 60 latest articles from 6 RSS feeds
   - Converts timestamps to Yangon time (UTC+6:30)
   - Renders HTML via Jinja2 template
   - Writes to index.html
6. **Commits & pushes** the updated index.html back to main branch
7. **GitHub Pages serves** the updated HTML automatically

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Site doesn't appear | Wait 2 minutes after enabling Pages |
| GitHub Actions doesn't run | Check "Actions" tab in repo, enable if needed |
| "Workflow file not found" | Ensure `.github/workflows/update-news.yml` exists in main branch |
| Feed fetch errors | Workflow tolerates 1-2 failed feeds; retries tomorrow |
